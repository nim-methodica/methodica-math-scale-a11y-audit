🔴 #28 — שדה קלט ללא תווית (הוסיפו aria-label)

לכל שדה-תשובה הוסיפו aria-label שמתאר מה להזין. דוגמה ל-.s18-answer-box:

<!-- לפני -->
<input class="s18-answer-box" id="s18-answer-input" type="text" dir="rtl">

<!-- אחרי -->
<input class="s18-answer-box" id="s18-answer-input" type="text" dir="rtl"
       aria-label="הקלידו את התשובה המספרית">

אם יש כיתוב נראה לפני השדה (<span id="s18-q">...</span>), עדיף לקשר אליו:

<input class="s18-answer-box" id="s18-answer-input" type="text" aria-labelledby="s18-q">

החילו על כל המופעים: 01-01 (1093,1211,1442) · 01-02 (464,744) · 01-03 (80) · 01-04 (444,643,756).

🟠 #13 — חיווי-פוקוס חלש (החליפו ב-:focus-visible אחיד)

כיום הרכיבים מסתמכים על border-color:#a51cff75 (סגול שקוף) או outline:none. הוסיפו כלל גלובלי

שמבטיח חיווי-פוקוס בולט לכל רכיב הניתן למיקוד:

/* תיקון גלובלי — להוסיף ל-styles.css */
*:focus-visible {
  outline: 3px solid #7100B9;   /* סגול כהה, ניגודיות ≥3:1 מול לבן */
  outline-offset: 2px;
  border-radius: 4px;
}
/* אם קיים outline:none על רכיבים אינטראקטיביים — להסירו או להחליפו בכלל לעיל */
.option-card:focus-visible,
.scale-input:focus-visible + .scale-marker {
  outline: 3px solid #7100B9;
  outline-offset: 2px;
}

הימנעו מ-outline:none ללא חלופה. :focus-visible מציג את החיווי רק בניווט-מקלדת (לא בעכבר),

כך שאין פגיעה בעיצוב למשתמשי-עכבר.

🟠 #26 — reflow/הגדלה (הקנבס הקבוע 1280×710)

המבנה הנוכחי מקבע קנבס ומקטין אותו כיחידה (transform: scale), כך שטקסט לא ניתן להגדלה עצמאית.

שתי גישות (לפי מורכבות שהצוות מוכן לקבל):

/* גישה א' (מינימלית) — לאפשר זום-טקסט עד 200% בלי חיתוך, גם אם נשאר scale:
   ודאו שמיכל-העל אינו חוסם גלילה אנכית, והסירו max-height קשיח על תיבות-טקסט. */
#app { overflow: auto; }
.hook-card-inner, .s7-card-q, .s18-answer-box { max-height: none; }

/* גישה ב' (נכונה לטווח-ארוך) — פריסה גמישה במקום קנבס קבוע:
   החליפו רוחב/גובה קבועים ב-min()/clamp() ואפשרו לתוכן לרדת שורה (wrap). */
#app { width: min(1280px, 100vw); height: auto; aspect-ratio: auto; }
.s39-cols, .option-cards { flex-wrap: wrap; }

בנוסף ודאו שתגית ה-viewport אינה חוסמת זום (היא תקינה כיום). מומלצת גישה ב' לעמידה מלאה ב-1.4.10.

חלונות-רמז, מלכודת-פוקוס במודל-הדיווח, ניתוב-פוקוס במעברי-מסך, צבעי נכון/שגוי) דורשים **בדיקה ידנית
