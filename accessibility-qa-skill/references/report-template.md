**5 עמודות**. סמל-החומרה פותח את עמודת "סוג הבעיה" כדי שהממיר יצבע את השורה.

## כללי כתיבת ממצא
- **5 העמודות:** (1) סמל-חומרה + `#מספר שם-בדיקה`; (2) מיקום מדויק `file:line` (או `target` של axe);
  (3) ציטוט מילולי קצר / מה נמצא; (4) המלצת תיקון קונקרטית הפותחת ב**פעולה מודגשת**; (5) מה הפתרון
  פותר + **הקריטריון בסוגריים** (למשל "WCAG 2.1.1 (A)").
- **בלי מיקום אין ממצא.** השתמש ב-`file:line` מ-scan.py, או ב-CSS-selector/`target` מ-axe.
- **כיסוי מלא — אסור לקצר:** כל מופע נרשם. רשימה ארוכה של אותו ממצא → שורה אחת עם כל המיקומים
  (`index.html:120, 145, 203`). אל תכתוב "ועוד"/"etc.".
- **המלצה החלטית אחת** בפעולה מפורשת ("החליפו ל-`#595959`", "הוסיפו מאזין keydown"). הטקסט-לתיקון
  בגרשיים/`code`.
- ניגודיות — **תמיד** ציין: היחס שנמדד, היחס הנדרש, זוג-הצבעים, וגודל-הגופן.
- **דיווח בלבד — אסור לערוך את קוד הלומדה.**
- **חובה:** כל 33 הבדיקות בטבלת-הכיסוי, כל אחת עם סטטוס (✅ / ממצא / ➖+סיבה). עקביות בין הטבלה
  לפירוט הקטגורי.

## render ל-HTML (`report.html`)
```bash
node "<skill>/scripts/md-to-html.js" "<report.md>"
```
הממיר (Node, ללא תלויות) מייצר HTML מעוצב: `dir="rtl" lang="he"`, צבעי-חומרה (🔴/🟠/🔵), טבלאות,
ועוגנים. מסור למשתמש גם את `report.md` וגם את `report.html`.
